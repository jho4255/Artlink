## 상세

### KI-1. 개발용 로그인이 운영에 열려 있음 🔴 (의도적 보류)
- **무슨 문제**: 비밀번호 없이 아무 계정으로 로그인되는 개발용 기능(`/api/auth/dev-login`, `/dev-users`)이 실서버에도 켜져 있음. 아무나 관리자가 될 수 있음.
- **왜 안 고침**: 카카오 OAuth 미승인 상태라 임시로 이걸 쓰는 중. 정식 로그인 도입할 때 같이 막기로 결정함.
- **고칠 때**: OAuth 붙이면서 `dev-login`/`dev-users`를 운영(production)에서 차단. (검증: curl로 401/404 확인)

### KI-2. 정원 초과 지원 미차단 🟡 → ✅ 수정됨 (2026-05-31)
- **무슨 문제였나**: 모집인원 1명짜리 공모에 2명이 지원해도 둘 다 접수됐음.
- **수정**: 지원 시 현재 지원자 수가 정원 이상이면 400("모집 인원이 마감되었습니다.")로 차단. (backend/src/routes/exhibition.ts apply 핸들러)
- **검증**: e2e `06-tier3-edge` 정원 테스트 통과(이제 진짜 차단), backend `known-issues-fixes.test.ts` 2케이스(초과 차단 / 여유 시 정상).

### KI-3. 삭제된 대상 수정요청 승인 시 500 🟢 → ✅ 수정됨 (2026-05-31)
- **무슨 문제였나**: 대상 갤러리/공모가 삭제됐는데 수정요청을 승인하면 "데이터 처리 중 오류"(500).
- **수정**: 승인 적용 전에 대상 존재를 확인 → 없으면 404("수정 대상을 찾을 수 없습니다…"), 요청 상태도 변경 안 함(대상 갱신 성공 후에만 APPROVED 처리). (backend/src/routes/approval.ts edit-request 핸들러)
- **검증**: curl 404 확인 + backend `known-issues-fixes.test.ts` 2케이스(대상없음 404·PENDING유지 / 정상 승인 반영).

### KI-4. 사업자 정보 빈칸 🟡
- **무슨 문제**: 푸터의 주소/사업자등록번호/통신판매신고번호/개인정보담당/대표번호가 비어 있음(법적 표시 의무).
- **고칠 때**: 코드 아님. `frontend/public/terms/company-info.txt`에 실제 값 입력.
