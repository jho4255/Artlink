## 상세

### KI-1. 개발용 로그인이 운영에 열려 있음 🔴 (의도적 보류)
- **무슨 문제**: 비밀번호 없이 아무 계정으로 로그인되는 개발용 기능(`/api/auth/dev-login`, `/dev-users`)이 실서버에도 켜져 있음. 아무나 관리자가 될 수 있음.
- **왜 안 고침**: 카카오 OAuth 미승인 상태라 임시로 이걸 쓰는 중. 정식 로그인 도입할 때 같이 막기로 결정함.
- **고칠 때**: OAuth 붙이면서 `dev-login`/`dev-users`를 운영(production)에서 차단. (검증: curl로 401/404 확인)

### KI-2. 정원 초과 지원 미차단 🟡
- **무슨 문제**: 모집인원 1명짜리 공모에 2명이 지원해도 둘 다 접수됨. 서버가 현재 지원자 수와 정원을 비교하지 않음.
- **실제로**: 갤러리가 정원보다 많은 지원을 받게 되어 운영에 혼선.
- **확인됨**: E2E 테스트가 정원 1명 공모에 2명 지원 → 2번째가 차단(400)돼야 하는데 통과(201)됨을 잡아냄. (`test.fail`로 표시 → 고치면 자동으로 초록불)
- **고칠 때**: Claude한테 **"공모 지원 시 현재 지원자 수가 정원 이상이면 400으로 막아줘"** (참고: backend/src/routes/exhibition.ts apply 핸들러)

### KI-3. 삭제된 대상 수정요청 승인 시 500 🟢
- **무슨 문제**: 갤러리/공모가 이미 삭제됐는데 그 수정요청을 승인하면 친절한 안내 없이 "데이터 처리 중 오류"(500). 정상 경로는 잘 동작.
- **고칠 때**: Claude한테 **"수정요청 승인 시 대상이 없으면 500 대신 '대상을 찾을 수 없습니다' 안내를 줘"** (참고: backend/src/routes/approval.ts edit-request 핸들러)

### KI-4. 사업자 정보 빈칸 🟡
- **무슨 문제**: 푸터의 주소/사업자등록번호/통신판매신고번호/개인정보담당/대표번호가 비어 있음(법적 표시 의무).
- **고칠 때**: 코드 아님. `frontend/public/terms/company-info.txt`에 실제 값 입력.
