## 🔧 고친 것 (문제별 — 무엇을 / 어떻게 확인했는지)

### 🟡 출시 직후 빨리 (요청 2~7)

- **문제 2 — 없는 주소 → 404 안내 페이지**
  - 잘못된 주소로 들어오면 흰 화면 대신 "페이지를 찾을 수 없습니다 + 홈으로 돌아가기" 버튼이 나옵니다.
  - 확인: 자동 브라우저로 엉뚱한 주소를 열어 안내가 뜨고, 홈 버튼이 작동하고, 정상 페이지는 그대로인 것까지 확인.

- **문제 3 — 없는 갤러리/공모/전시 → "찾을 수 없습니다" 안내+목록 버튼**
  - 삭제됐거나 없는 상세 페이지를 열면 안내문과 "목록으로" 버튼이 즉시 나옵니다. (이전엔 빈 화면)
  - 확인: 없는 번호(999999) 3종 모두 안내+버튼 표시, 실제 갤러리는 정상 표시.

- **문제 4 — 목록 불러오기 실패 시 오류 안내+다시 시도**
  - 인터넷/서버 문제로 목록을 못 불러오면 "불러오지 못했습니다 + 다시 시도" 버튼이 나옵니다.
  - 확인: **백엔드를 일부러 꺼서** 3개 목록 페이지 모두 안내가 뜨는 걸 보고, 다시 켜서 목록이 정상 복구되는 것까지 확인.

- **문제 5 — 외부 부품(axios·multer) 보안 업데이트**
  - 통신 부품 axios(1.13→1.16), 파일 부품 multer(2.1.0→2.1.1)를 안전한 버전으로 올렸습니다.
  - 확인: 보안 검사에서 두 부품 경고 사라짐, 파일 업로드 정상, 백엔드 313개·프론트 35개 검사 통과.

- **문제 6 — 파일 업로드 실제 형식(MIME) 검사**
  - 파일 이름만 보던 걸, 실제 파일 형식까지 확인하도록 했습니다. 이름만 .pdf로 위장한 실행파일 등은 거부됩니다.
  - 확인: 정상 PDF/한글파일/이미지는 통과, .pdf로 위장한 실행파일·HTML은 차단되는 것 확인.

- **문제 7 — 지원 상태 거꾸로 되돌리기 차단**
  - 접수→검토중→수락/거절 순서만 허용하고, "수락"을 다시 "접수"로 되돌리는 등 역행을 막았습니다.
  - 확인: 정상 전진은 되고, 되돌리기는 거부(400)되며 상태가 그대로 유지되는 것 확인.

### 🟢 천천히 (요청 8~13, 15, 16)

- **문제 8 — 잘못된 값 입력 시 500 대신 400(친절한 거절)**
  - 숫자 자리에 글자가 오는 등 잘못된 입력에 "서버 오류" 대신 "잘못된 요청"으로 응답합니다.
  - 확인: 글자 입력 → 400(이전 500), 정상 입력 → 정상 동작. 위장 업로드 거부도 깔끔한 400으로 바뀜.

- **문제 9 — 알림 드롭다운 화면 넘침 방지(하드닝)**
  - 알림창이 화면을 넘지 않도록 폭 제한을 추가했습니다.
  - 확인: 모바일·데스크톱 모두 알림창이 화면 안에 들어옴.
  - 참고: 알아보니 모바일은 원래 별도의 꽉 찬 알림창을 써서 "넘침"이 실제로는 안 생기고 있었습니다. 이번 변경은 안전장치 성격입니다.

- **문제 10 — 작은 버튼 터치 영역 확대**
  - 리뷰 수정/삭제 버튼을 손가락으로 누르기 쉬운 크기(44×44)로 키웠습니다(아이콘 모양은 그대로).
  - 확인: 해당 버튼들이 모두 44×44로 측정됨.

- **문제 11 — 이미지 설명(alt) 추가**
  - 갤러리 사진·리뷰 사진·프로필·작품 이미지에 설명을 넣어 화면낭독기가 읽을 수 있게 했습니다.
  - 확인: 갤러리 카드·리뷰·작품 이미지에 설명이 들어간 것 확인.
  - (작업 중 빌드 오류 1건을 만들었다가 빌드 검사로 잡아 바로 고쳤습니다 — 커밋 fb08341.)

- **문제 12 — 인스타 피드 실패 시 안내**
  - 인스타 연동이 실패하면 빈 화면 대신 "Instagram 피드를 불러오지 못했습니다" 안내가 나옵니다.
  - 확인: 일부러 잘못된 인스타 연결을 만들어 안내가 뜨는 걸 보고, 인스타 안 쓰는 갤러리엔 안내가 안 뜨는 것까지 확인. 검사용으로 바꾼 데이터는 원상복구함.

- **문제 13 — 로그 파일 경로 안전 처리**
  - 보안 검사가 경고하던 부분을, 정해진 파일만 쓰도록 구조를 바꿔 경고를 없앴습니다.
  - 확인: 보안 검사(semgrep) 경고 0건, 로그 기록 정상 동작.

- **문제 15 — 커스텀 질문 "최대 선택 수" 서버 검증**
  - 옵션 2개인데 "최대 5개 선택" 같은 모순 설정을 서버에서도 막도록 했습니다(이전엔 화면에서만 막음).
  - 확인: 모순 설정 → 거부(400), 정상 설정 → 통과.

- **문제 16 — 내 정보(/me)에 프로필 사진 포함**
  - 프로필 사진을 바꾼 뒤 정보를 다시 불러와도 사진이 사라지지 않도록 응답에 사진 정보를 포함시켰습니다.
  - 확인: 사진 설정 후 /me 응답에 사진 주소가 정상 포함됨. 검사용 데이터는 원상복구함.
